# Тестирование модуля "Избранное вакансий"

## Предварительные условия

1. Зарегистрируйтесь и войдите как пользователь (получите access token)
2. Создайте профиль (обязательно!)
3. Создайте хотя бы одну вакансию (как ADMIN)

## Примеры запросов в Postman

### 1. Добавление вакансии в избранное

**POST** `http://localhost:3000/api/jobs/:id/favorite`

**Headers:**
```
Authorization: Bearer <your_access_token>
```

**Успешный ответ (200):**
```json
{
  "message": "Вакансия добавлена в избранное",
  "favoriteJobsCount": 1
}
```

**Возможные ошибки:**
- `400` - Вакансия уже добавлена в избранное
- `400` - Нельзя добавить неактивную вакансию
- `404` - Вакансия не найдена
- `404` - Профиль не найден (создайте профиль)

---

### 2. Получение списка избранных вакансий

**GET** `http://localhost:3000/api/jobs/favorites`

**Headers:**
```
Authorization: Bearer <your_access_token>
```

**Успешный ответ (200):**
```json
{
  "total": 2,
  "favoriteJobs": [
    {
      "_id": "677e8e9a1234567890abcdef",
      "title": "Senior Frontend Developer",
      "company": "TechCorp",
      "direction": "IT",
      "level": "Senior",
      "workFormat": "Hybrid",
      "location": "Москва",
      "salary": {
        "min": 200000,
        "max": 300000,
        "currency": "RUB"
      },
      "isActive": true,
      "createdAt": "2026-01-29T10:00:00.000Z"
    }
  ]
}
```

**Особенности:**
- Показываются только активные вакансии
- Удаленные вакансии автоматически исключаются

---

### 3. Удаление вакансии из избранного

**DELETE** `http://localhost:3000/api/jobs/:id/favorite`

**Headers:**
```
Authorization: Bearer <your_access_token>
```

**Успешный ответ (200):**
```json
{
  "message": "Вакансия удалена из избранного",
  "favoriteJobsCount": 1
}
```

**Возможные ошибки:**
- `400` - Вакансия не найдена в избранном
- `404` - Профиль не найден

---

## Сценарии для тестирования

### ✅ Базовый flow:

1. Создайте профиль (`POST /api/profile`)
2. Получите список вакансий (`GET /api/jobs`)
3. Добавьте вакансию в избранное (`POST /api/jobs/:id/favorite`)
4. Проверьте список избранных (`GET /api/jobs/favorites`)
5. Удалите вакансию из избранного (`DELETE /api/jobs/:id/favorite`)

### ✅ Проверка дублирования:

1. Добавьте вакансию в избранное
2. Попробуйте добавить её снова → ошибка 400

### ✅ Проверка без профиля:

1. Зарегистрируйтесь, но НЕ создавайте профиль
2. Попробуйте добавить вакансию → ошибка 404 "Создайте профиль"

### ✅ Проверка неактивной вакансии:

1. Как ADMIN, создайте вакансию
2. Деактивируйте её (`DELETE /api/jobs/:id`)
3. Попробуйте добавить в избранное → ошибка 400

### ✅ Автоматическая фильтрация:

1. Добавьте несколько вакансий в избранное
2. Деактивируйте одну из них (как ADMIN)
3. Запросите список избранных → деактивированная вакансия не отображается

## Структура данных в MongoDB

После добавления в избранное, в документе Profile появится массив:

```javascript
{
  "_id": ObjectId("..."),
  "userId": ObjectId("..."),
  "name": "Иван Иванов",
  "direction": "IT",
  "level": "Middle",
  // ... другие поля
  "favoriteJobs": [
    ObjectId("677e8e9a1234567890abcdef"),
    ObjectId("677e8e9a1234567890abcd00")
  ]
}
```

---

## Важные замечания

⚠️ **Профиль обязателен:** Без созданного профиля вы не сможете добавлять вакансии в избранное.

⚠️ **Только активные:** Можно добавить только активные вакансии (`isActive: true`).

✅ **Автоматическая очистка:** При получении списка избранного неактивные и удаленные вакансии автоматически фильтруются.

✅ **Популяция данных:** В списке избранных возвращаются полные данные о вакансиях и создателе.
